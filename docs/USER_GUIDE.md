## Вступ

DiskDataAnalyzer - потужний інструмент для аналізу та управління даними на дисках Windows. Підтримує:

- 🤖 LLM-класифікацію файлів (Claude, OpenAI, Ollama, KiroAI)
- 📁 Візуалізацію структури директорій
- 🔄 Порівняння папок
- 🗑️ Виявлення та безпечне видалення сміття
- 📊 Детальну аналітику використання диску

### Для кого цей інструмент?

- Системні адміністратори
- Розробники
- Користувачі, які хочуть оптимізувати використання диску
- Всі, хто потребує детального аналізу файлової системи

## Встановлення

### Windows (Рекомендовано)

**Варіант 1: Готовий executable (найпростіше)**

1. Завантажити останній реліз: https://github.com/OniXinO/DiskDataAnalyzer/releases
2. Розпакувати `DiskDataAnalyzer-vX.X.X-Windows.zip`
3. Запустити `DiskDataAnalyzer-vX.X.X.exe`

**Варіант 2: З вихідного коду**

```bash
# Клонувати репозиторій
git clone https://github.com/OniXinO/DiskDataAnalyzer.git
cd DiskDataAnalyzer

# Встановити залежності
pip install -r requirements.txt

# Запустити
python run.py
```

### Системні Вимоги

- **OS:** Windows 7/8/10/11
- **Python:** 3.9+ (якщо запуск з коду)
- **RAM:** 512 MB мінімум, 2 GB рекомендовано
- **Диск:** 50 MB для програми

## Швидкий Старт

### Перший Запуск

1. Запустити `run.py` або `.exe` файл
2. Відкриється головне вікно з 4 вкладками
3. Вибрати потрібну функцію

### 5-хвилинний Туторіал

**Сценарій: Знайти та видалити сміттєві файли**

1. Відкрити вкладку **"Junk Detector"**
2. Натиснути **"Select Folder"** → вибрати папку для сканування
3. Переконатись що всі типи сміття відмічені
4. Натиснути **"Scan"**
5. Переглянути результати в таблиці
6. Натиснути **"Safe Delete"** → підтвердити видалення

**Результат:** Сміттєві файли безпечно видалено, місце звільнено!

## GUI Режим

### Вкладка 1: File Classifier

**Призначення:** Автоматична класифікація файлів за допомогою LLM.

#### Як Використовувати

1. **Вибрати папку:**
   - Натиснути "Select Folder"
   - Вибрати директорію для аналізу

2. **Вибрати LLM провайдер:**
   - **Claude** - найточніший, потребує API ключ
   - **OpenAI** - швидкий, потребує API ключ
   - **Ollama** - локальний, безкоштовний
   - **KiroAI** - альтернативний провайдер

3. **Класифікувати:**
   - Натиснути "Classify"
   - Дочекатись завершення (прогрес-бар)

4. **Переглянути результати:**
   - Таблиця з 5 колонками:
     - File - ім'я файлу
     - Category - категорія
     - Subcategory - підкатегорія
     - Description - опис
     - Confidence - впевненість (0.0-1.0)

5. **Експортувати:**
   - "Export CSV" - для Excel
   - "Export JSON" - для програмної обробки

#### Налаштування API Ключів

**Claude:**
```bash
set ANTHROPIC_API_KEY=your_key_here
```

**OpenAI:**
```bash
set OPENAI_API_KEY=your_key_here
```

**Ollama:**
- Встановити Ollama: https://ollama.ai
- Запустити локальний сервер: `ollama serve`

#### Кеш

Результати класифікації зберігаються в `classification_cache.db` для швидкості. Кеш автоматично інвалідується при зміні файлів.

### Вкладка 2: Directory Tree

**Призначення:** Візуалізація ієрархічної структури директорій.

#### Як Використовувати

1. **Вибрати папку:**
   - "Select Folder" → вибрати директорію

2. **Налаштувати фільтри:**
   - **Ignore patterns:** `.git, __pycache__, node_modules`
   - **Max depth:** Unlimited, 1, 2, 3, 5, 10

3. **Побудувати дерево:**
   - "Build Tree"
   - Дочекатись завершення

4. **Переглянути:**
   - 📁 Папки з іконками
   - 📄 Файли з розміром
   - Розгортати/згортати гілки

5. **Статистика:**
   - Files count - кількість файлів
   - Folders count - кількість папок
   - Total size - загальний розмір

6. **Експорт:**
   - "Export Text" - Unicode дерево (├── └── │)

#### Приклад Виводу

```
📁 project/
├── 📁 src/
│   ├── 📄 main.py (2.5 KB)
│   └── 📄 utils.py (1.2 KB)
├── 📁 tests/
│   └── 📄 test_main.py (3.1 KB)
└── 📄 README.md (5.4 KB)
```

### Вкладка 3: Folder Compare

**Призначення:** Порівняння двох папок для знаходження відмінностей.

#### Як Використовувати

1. **Вибрати папки:**
   - "Select Folder 1" → перша папка
   - "Select Folder 2" → друга папка

2. **Налаштувати опції:**
   - **Recursive:** включити підпапки
   - **Use Hash:** порівняння за MD5 (точніше) або розміром/датою (швидше)

3. **Порівняти:**
   - "Compare"
   - Дочекатись завершення

4. **Переглянути результати:**
   - **✓ Identical** (зелений) - файли ідентичні
   - **≠ Different content** (помаранчевий) - різний вміст
   - **→ Only in folder 1** (синій) - тільки в першій папці
   - **← Only in folder 2** (червоний) - тільки в другій папці

5. **Статистика:**
   - Identical - кількість ідентичних
   - Different - кількість різних
   - Only in 1 - тільки в першій
   - Only in 2 - тільки в другій

6. **Експорт:**
   - "Export Report" - текстовий звіт

#### Випадки Використання

- **Синхронізація:** Знайти файли для копіювання
- **Бекап:** Перевірити повноту резервної копії
- **Міграція:** Порівняти старі та нові директорії
- **Дедуплікація:** Знайти дублікати між папками

---

### Вкладка 4: Junk Detector

**Призначення:** Виявлення та безпечне видалення сміттєвих файлів.

#### Як Використовувати

1. **Вибрати папку:**
   - "Select Folder" → директорія для сканування

2. **Вибрати типи сміття:**
   - ☑ **Temp files** - тимчасові файли (*.tmp, *.cache)
   - ☑ **Backup files** - бекапи (*.bak, *.backup)
   - ☑ **Old backups** - старі бекапи (>30 днів)
   - ☑ **Duplicates** - дублікати (за MD5)
   - ☑ **Empty folders** - порожні папки

3. **Налаштувати:**
   - **Recursive:** включити підпапки

4. **Сканувати:**
   - "Scan"
   - Дочекатись завершення

5. **Переглянути результати:**
   - Таблиця з 3 колонками:
     - File - шлях до файлу
     - Type - тип сміття
     - Size - розмір

6. **Статистика:**
   - Total files found - знайдено файлів
   - Total size - загальний розмір

7. **Безпечне видалення:**
   - "Safe Delete"
   - Підтвердити в діалозі
   - ⚠️ **Операція незворотна!**

#### Захист Системних Файлів

Whitelist автоматично захищає:
- `System32*`
- `Windows*`
- `Program Files*`
- `*.sys`, `*.dll`
- `boot*`, `ntldr`, `bootmgr`

#### Рекомендації

- **Перший раз:** Сканувати без видалення, переглянути результати
- **Бекап:** Створити бекап перед видаленням важливих даних
- **Обережно:** Перевірити список перед видаленням

## CLI Режим

### Базовий Аналіз Диску

```bash
# Аналіз одного диску
python src/core/analyze_disk.py C:

# Аналіз всіх дисків
python src/core/analyze_disk.py --all

# Вказати директорію для звітів
python src/core/analyze_disk.py D: --report-dir O:/my_reports
```

### Експорт Звітів

```bash
# JSON експорт
python src/core/analyze_disk.py C: --export json --output report.json

# CSV експорт
python src/core/analyze_disk.py C: --export csv --output report.csv

# HTML експорт
python src/core/analyze_disk.py C: --export html --output report.html
```

### Пошук Великих Файлів

```bash
python src/core/large_files.py C: --min-size 100MB --limit 50
```

## Поширені Питання

### Q: Чи безпечно видаляти файли через Junk Detector?

**A:** Так, якщо дотримуватись рекомендацій:
- Завжди переглядайте список перед видаленням
- Whitelist захищає системні файли
- Створюйте бекап важливих даних
- Операція незворотна - будьте обережні

### Q: Скільки коштує використання LLM класифікації?

**A:** Залежить від провайдера:
- **Ollama:** Безкоштовно (локальний)
- **Claude/OpenAI:** Платно (API ключ)
- **KiroAI:** Залежить від тарифу

### Q: Чому класифікація повільна?

**A:** 
- Перший раз: Класифікація через LLM займає час
- Наступні рази: Використовується кеш (швидко)
- Рекомендація: Використовуйте Ollama для великих наборів

### Q: Як працює кеш класифікації?

**A:**
- Результати зберігаються в SQLite БД
- Кеш інвалідується при зміні файлу (розмір/дата)
- Локація: `classification_cache.db`

### Q: Чи можна використовувати на мережевих дисках?

**A:** Так, але:
- Може бути повільніше
- Потрібні права доступу
- Рекомендується локальні диски

## Усунення Проблем

### Програма не запускається

**Проблема:** Помилка при запуску `.exe`

**Рішення:**
1. Перевірити Windows Defender (може блокувати)
2. Запустити від імені адміністратора
3. Перевірити наявність Python (якщо з коду)

### Помилка "Permission Denied"

**Проблема:** Немає доступу до файлів/папок

**Рішення:**
1. Запустити від імені адміністратора
2. Перевірити права доступу до папки
3. Закрити файли, які використовуються іншими програмами

### LLM класифікація не працює

**Проблема:** Помилка при класифікації

**Рішення:**
1. Перевірити API ключ (змінна середовища)
2. Перевірити інтернет з'єднання (Claude/OpenAI)
3. Перевірити Ollama сервер (локальний)
4. Переглянути логи для деталей

### Тести не проходять

**Проблема:** `python -m unittest discover` падає

**Рішення:**
1. Встановити всі залежності: `pip install -r requirements.txt`
2. Перевірити Python версію (3.9+)
3. Запустити з кореня проєкту

### GUI зависає

**Проблема:** Інтерфейс не відповідає

**Рішення:**
1. Дочекатись завершення операції (прогрес-бар)
2. Зменшити обсяг даних для обробки
3. Перезапустити програму
