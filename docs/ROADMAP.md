## 📅 ФАЗА 1: Покращення CLI (1 тиждень) → v0.2.0

### Пріоритет: P0 (Критично)

### 1.1 Оптимізація продуктивності
**Час:** 2 дні

**Проблема:** Аналіз великих дисків (>100GB) занадто повільний

**Рішення:**
- Багатопотоковість для сканування директорій
- Кешування результатів
- Прогрес-бар в CLI
- Можливість зупинки/відновлення

**Файли:**
- `src/core/scanner.py` (новий) - багатопотоковий сканер
- `src/core/cache.py` (новий) - кешування результатів
- `src/core/progress.py` (новий) - прогрес-бар

**Тести:**
- `tests/test_scanner.py`
- `tests/test_cache.py`

### 1.2 Розширення аналізу

**Додати:**

## 📝 Правила розробки

### Git workflow:
- Гілка `dev` для розробки
- Гілка `master` для релізів
- Feature branches: `feature/gui-main-window`
- Bugfix branches: `bugfix/symlink-crash`

### Коміти:
- Conventional Commits (feat/fix/docs/test/refactor)
- Один коміт = одна задача
- Детальні повідомлення

### Тестування:
- TDD підхід
- Unit tests для всіх функцій
- Integration tests для workflows
- Manual testing перед релізом

### Code review:
- Після кожної фази
- Використання `/code-reviewer` skill
